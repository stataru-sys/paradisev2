# Work System Expansion — Design Plan

> Keeper-документ. Не код, не сцены — только дизайн, баланс и план слайсов для coder'а.

## 1. Цель изменения

Сейчас приложение «Работа» = одна мини-игра (сортировка слов). Игрок кликает «Работа» → сразу в сортировку. Это ок для Run 1, но для Run 2+ нужно разнообразие.

**Что меняем:**
- Добавляем Work Hub — экран выбора между разными видами работы.
- Текущая сортировка становится одной из опций, а не единственной.
- Добавляем новые мини-игры с разной механикой, риском и наградой.
- Связываем с апгрейдами и ИИ-агентами.

**Почему это важно:**
- Разнообразие геймплея — чтобы не было «опять сортировка».
- Стратегический выбор — игрок решает, рисковать или действовать стабильно.
- Пространство для апгрейдов и агентов — каждая мини-игра получает свои усилители.
- Сатира — каждая мини-игра пародирует свой аспект офисной жизни 2033.

---

## 2. Новый UX приложения «Работа»

```
Рабочий стол → клик «Работа»
  → Work Hub (список доступных работ)
    → клик «Начать» на конкретной работе
      → Мини-игра
        → Результат работы (деньги / ошибки / энергия / комментарий)
          → Возврат в Work Hub  или  Рабочий стол (если энергия кончилась)
```

**Work Hub — экран выбора:**
- Список карточек работ (VBoxContainer в ScrollContainer).
- Каждая карточка: название, описание, награда, энергия, риск, кнопка «Начать».
- Заблокированные работы — серые, с замком и подсказкой «Откроется позже».
- Если энергия = 0 — все кнопки «Начать» заблокированы.

**Результат работы — overlay/панель после мини-игры:**
- Заработано: +X$
- Потеряно на ошибках: -Y$ (если были)
- Энергия: -Z
- Качество: «Отлично» / «Сойдёт» / «Провал»
- Сатирический комментарий
- Кнопка «Вернуться к работе» → Work Hub
- Кнопка «На рабочий стол» → Desktop

## 4. Подробное описание каждой мини-игры

### 4.1. Сортировка задач (`sorting`)

**Уже реализована** в `scripts/run1/WorkProgram.gd`. Оставить как есть, только:
- Вынести `WORDS_BY_CATEGORY` в Dictionary на 5 категорий (сейчас 3).
- Добавить кнопку «Закончить работу» до того как все слова разобраны — игрок сам решает когда остановиться.
- После завершения показывать результат (пока finish_panel просто закрывает программу).

**Сатира:** «Сортируешь слова, которые мог бы отсортировать любой скрипт за 0.3 секунды. Но ты — человек. Тебе платят за страдание.»

**Связь с апгрейдами:**
- `monitor`: +$5 за полную категорию (уже есть).
- `autoclicker`: кнопка «Решить 1 карточку» (уже есть).
- `coffee_machine`: +1 категория без штрафа за ошибку.
- `ergonomic_mouse`: +2 слова в пуле (больше потенциальной награды).

**Связь с агентами:**
- `intern_gpt`: решает 1 карточку (уже есть, 25% ошибка).
- `bugdav`: может переложить уже правильно отсортированное слово (создаёт хаос-эффект).

### 4.2. Корпоративная почта (`mail_sort`)

**Сатира:** «Письма от тех, кто тоже не знает, зачем работает. Твой долг — рассортировать их так, чтобы никто ничего не заметил.»

**Правила:**
- Игроку показывается 5–7 писем по одному.
- Каждое письмо — короткий текст + имя отправителя.
- Нужно выбрать папку: «Срочно», «Игнорировать», «Делегировать», «Ответить шаблоном».

**Пример письма:**

```
От: Михаил, отдел синергии
«Коллеги, предлагаю синхронизироваться по вчерашнему
созвону. Когда всем удобно?»

[Срочно] [Игнорировать] [Делегировать] [Шаблон]
```

- Правильный ответ: «Игнорировать» (байт на созвон → ещё созвон).
- Если выбрать «Срочно» → ошибка, штраф -$2.
- Если выбрать «Шаблон» → нормально, +$3.

**Типы писем:**
- Обычное рабочее (1 правильная папка, 1 нейтральная, 2 ошибочные).
- Письмо-ловушка: выглядит как спам, но от начальника — если проигнорировать, штраф.
- Письмо от AI-агента: сам себя поставил в копию, отвечать бессмысленно.
- Приглашение на митинг: любое действие кроме «Игнорировать» = +1 встреча в календаре.

**Награда:**
- Правильная папка: +$3–5
- Нейтральная папка: +$1–2
- Ошибочная папка: -$2–3
- Пропущено важное (начальник): -$5

**UI:**
- Верх: письмо (текст + отправитель).
- Низ: 4 кнопки-папки.
- Прогресс: «Письмо 3/6».

**Связь с апгрейдами:**
- `reply_templates`: +1 нейтральная папка становится правильной (шаблоны ответов).
- `spam_filter`: одно письмо-ловушка подсвечивается.

**Связь с агентами:**
- `mail_demon`: авто-сортирует 1 письмо. 30% шанс перепутать «Срочно» и «Игнорировать».

**Сложность реализации:** S
- 1 сцена + 1 скрипт.
- Пул писем — Dictionary в скрипте (15–20 писем).
- Никакого drag-drop, только кнопки.
- Результат — overlay как в сортировке.

### 4.3. Созвон без смысла (`meeting`)

**Сатира:** «Дожить до конца созвона живым, не став ответственным — искусство 2033 года.»

**Правила:**
- Игрок «участвует» в созвоне. Каждые 1.5–2.5 секунды появляется реплика от участника.
- Игрок должен выбрать реакцию за ~3 секунды (таймер).
- Если не выбрать — авто-молчание.
- Длительность: 10–12 реплик.

**Реплики и правильные реакции:**

| Реплика | Правильно | Нейтрально | Ошибка |
|---|---|---|---|
| «Давайте синхронизируемся» | Кивнуть | Молчать | Согласиться |
| «Коллеги, я без камеры» | Молчать | Кивнуть | Предложить follow-up |
| «А кто шарит экран?» | Молчать | Кивнуть | Согласиться |
| «Предлагаю вынести в отдельный созвон» | Кивнуть | Молчать | Согласиться |
| «У меня через 2 минуты следующий» | Молчать | Кивнуть | Предложить follow-up |
| «Нужен волонтёр» | Молчать | Молчать | ВСЁ (станешь ответственным) |

**Реакции (4 кнопки):**
- «Кивнуть»
- «Сказать „согласен“»
- «Предложить follow-up»
- «Молчать»

**Риск «Назначили ответственным»:**
- Если на «Нужен волонтёр» не промолчать — игрок становится ответственным.
- Штраф: -$5 в конце созвона + запись в EventLog.

**Награда:**
- Правильная реакция: +$2
- Нейтральная: +$1
- Ошибка: -$1
- Стал ответственным: -$5
- Дожил до конца: базовый бонус +$5

**UI:**
- Верхняя половина: лица участников (цветные квадраты) + реплика в центре.
- Нижняя половина: 4 кнопки реакций.
- Таймер-бар на каждой реплике (убывающая полоска).

**Связь с апгрейдами:**
- `anti_shame_mic`: ошибка реакции стоит -$0 вместо -$1 (микрофон выключен — никто не слышал).
- `second_monitor`: таймер на реакцию +1 секунда (можно подумать).

**Связь с агентами:**
- `meeting_avatar`: агент ходит на созвон вместо игрока — даёт фиксированный доход $10 без риска, но 15% шанс что агента назначат ответственным и он потратит 1 доп. энергию.

**Сложность реализации:** M
- 1 сцена + 1 скрипт.
- Таймер-механика.
- Пул из 15–20 реплик с правильными реакциями.
- UI таймер-бара.

### 4.4. Фикс багов (`bugfix`)

**Сатира:** «Баг — это фича, которую ещё не переименовали. А переименовать — твоя работа.»

**Правила:**
- Показывается 3 бага + 3 фикс-комментария (перемешаны).
- Игрок сопоставляет баг с правильным фиксом (drag-drop или клик-клик).
- Если ошибся — появляется четвёртый баг.
- Если багов > 5 — провал.
- Цель: закрыть все баги с минимумом ошибок.

**Пример:**

| Баг | Правильный фикс | Ловушки |
|---|---|---|
| «Кнопка исчезает после клика» | «Проверить visible в _on_pressed» | «Переименовать кнопку», «Удалить сцену», «Сказать что фича» |
| «Энергия не тратится» | «Добавить spend_energy в метод» | «Увеличить max_energy», «Перезагрузить ноут» |
| «Агент продаёт сообщения» | «Отозвать API-ключ» | «Купить второй ключ», «Обновить драйвер» |

**Награда:**
- Правильный фикс: +$5
- Ошибка: новый баг (доп. раунд), -$1
- Все 3 закрыты без ошибок: бонус +$5
- Провал (>5 багов): $0, «Передано в аутсорс»

**UI:**
- Слева: список багов (VBoxContainer).
- Справа: список фиксов (VBoxContainer).
- Игрок кликает баг → кликает фикс → проверка.
- Прогресс: «Багов: 3 → 1».

**Связь с апгрейдами:**
- `stackoverflow_premium`: один неправильный фикс исключается из пула (подсвечивается красным).
- `second_monitor`: можно видеть на 1 баг больше одновременно.

**Связь с агентами:**
- `bugdav`: предлагает фикс для одного бага. 40% шанс что предложит неправильный. Если правильно — +$5, если нет — +2 бага.

**Сложность реализации:** S
- 1 сцена + 1 скрипт.
- Пул из 10–12 багов с фиксами.
- Механика сопоставления (клик-клик проще чем drag-drop).

### 4.5. Генерация отчёта (`report`)

**Сатира:** «Никто не читает отчёты. Но все смотрят на графики. Дай им графики.»

**Правила:**
- Игрок собирает отчёт из 4 блоков:
  1. График (идёт вверх / вниз / хаотичный)
  2. Метрика (реальная / придуманная / страшная)
  3. Вывод (оптимистичный / пессимистичный / бессмысленный)
  4. Прогноз (радужный / тревожный / «нужны доп. данные»)

- Каждый блок имеет скрытое «качество»:
  - Нормальный (60% шанс) — даёт очки.
  - Пустой (20%) — 0 очков, но не вредит.
  - Опасный (20%) — звучит гениально, но может взорваться (20% шанс штрафа).

- Игрок выбирает из 3 вариантов на каждый блок (не зная точного качества).
- После сборки отчёт «оценивается» — показывается итоговая убедительность.

**Награда:**
- Зависит от убедительности (суммы «очков»):
  - 8–10 очков: «Гениальный отчёт» → +$25–35
  - 5–7 очков: «Сойдёт» → +$10–20
  - 0–4 очка: «Кто это писал?» → +$0–5
- Опасный блок сработал: -$8 (списание премии)

**UI:**
- 4 секции по очереди (шаг 1/4, шаг 2/4...).
- В каждой секции 3 варианта на выбор.
- После выбора всех 4 — экран оценки с анимацией «проверки».

**Связь с апгрейдами:**
- `graph_generator`: один «пустой» блок заменяется на «нормальный» (всегда видно какой).
- `bullshit_detector`: «опасный» блок подсвечивается жёлтым (но всё ещё можно выбрать).

**Связь с агентами:**
- `report_shaman`: авто-генерирует отчёт. 60% шанс «Гениальный отчёт», 30% «Сойдёт», 10% «Полный провал» (0$ + -$5 за конфуз). Не тратит энергию.

**Сложность реализации:** M
- 1 сцена + 1 скрипт.
- Пул вариантов для каждого блока (9–12 текстов).
- Система скрытого качества + подсчёт очков.
- Пошаговый UI.

### 4.6. Проверка AI-агента (`ai_check`)

**Сатира:** «Твой агент уже всё сделал. Осталось проверить, не добавил ли он туда котиков и не удалил ли базу.»

**Правила:**
- Показывается 8–10 карточек «результатов работы AI-агента».
- Каждая карточка — короткое описание того, что сделал агент.
- Игрок нажимает: «Принять» / «Переделать» / «Списать на стажёра».
- Часть карточек — очевидно неправильные.
- Часть — спорные (выглядят неправильно, но по факту правильно).

**Типы карточек:**

| Карточка | Правильное действие | Почему |
|---|---|---|
| «Оптимизировал базу — удалил дубликаты» | Принять | Агент прав |
| «Переименовал все переменные в x1, x2, x3...» | Переделать | Агент сошёл с ума |
| «Отправил отчёт клиенту без твоего согласия» | Переделать | Нарушение протокола |
| «Добавил тёмную тему потому что 'так красивее'» | Принять | Не просили, но ок |
| «Удалил неиспользуемые ассеты — игра стала весить на 50MB меньше» | Принять | Реально полезно |
| «Подписался на все рассылки от твоего имени» | Переделать | Спам-апокалипсис |
| «Сказал клиенту что мы «почти закончили» (осталось 3 месяца)» | Списать на стажёра | Классика |
| «Сгенерировал 500 строк комментариев к коду» | Принять | Бесполезно, но не вредно |

**Награда / штраф:**
- Правильное действие (Принять хорошее / Переделать плохое): +$2
- Ошибка (Принять плохое): -$3 (последствия вранья)
- Ошибка (Переделать хорошее): -$1 (потеря времени)
- «Списать на стажёра»: всегда нейтрально (+$1), но можно только 3 раза.
- Доп. бонус: если все плохие найдены → +$5 «Бдительность».

**UI:**
- Карточка по центру: текст результата.
- 3 кнопки внизу.
- Счётчик «Списать на стажёра: 2/3».
- Прогресс: «Карточка 5/10».

**Связь с апгрейдами:**
- `ai_whisperer`: одна «спорная» карточка подсвечивается (видно, что она правильная).
- `intern_manual`: +1 использование «Списать на стажёра» (4 вместо 3).

**Связь с агентами:**
- `checker_gpt`: проверяет 3 карточки за тебя. 70% правильно, 20% нейтрально, 10% ошибка.
- Все существующие агенты могут генерировать карточки с историями из своих побочек.

**Сложность реализации:** M
- 1 сцена + 1 скрипт.
- Пул из 20–25 карточек.
- Три действия с разными последствиями.
- Интеграция с существующими агентами (ссылки на их побочки).

### 4.7. Фриланс-заказ (`freelance`)

**Сатира:** «Твоя ставка — $50/час. Клиент предлагает $15 фиксированно. Ты соглашаешься.»

**Правила:**
- Показывается 3 заказа (выбор из пула).
- Игрок выбирает один.
- После выбора — короткая «проверка качества» (1 клик).
- Результат зависит от:
  - Сложности заказа.
  - Шанса доп. правок.
  - Шанса токсичного клиента.
  - Шанса бонуса (клиент доволен → чаевые).

**Примеры заказов:**

| Заказ | Сложность | Базовая награда | Шанс правок | Шанс токсика | Шанс бонуса |
|---|---|---|---|---|---|
| «Дизайн логотипа для кота» | Лёгкая | $8 | 10% | 5% | 20% |
| «Сверстать лендинг за вечер» | Средняя | $15 | 30% | 20% | 15% |
| «AI-бот для знакомств за 3 дня» | Высокая | $25 | 60% | 40% | 25% |
| «Интеграция с API (документации нет)» | Адская | $40 | 80% | 60% | 10% |

**Механика:**
- Выбор заказа → кнопка «Выполнить» → быстрый результат:
  - «Сдано с первого раза» (+полная награда)
  - «Правки» (+50% награды)
  - «Токсичный клиент» (-30% награды, +1 доп. энергия)
  - «Бонус» (+150% награды, редкое)
- Это быстрая одноэкранная операция.

**UI:**
- 3 карточки заказов в ряд.
- При наведении — детали.
- Кнопка «Взять заказ» под каждой.
- После взятия → мини-экран с одной кнопкой «Выполнить» и полоской «качества» (авто-результат).

**Связь с апгрейдами:**
- `client_shield`: шанс токсичного клиента -15%.
- `fast_delivery`: шанс правок -15%.

**Связь с агентами:**
- `freelance_proxy`: общается с клиентом вместо игрока. Шанс токсичного клиента -30%, но 15% шанс что агент согласится на скидку (награда -30%).

**Сложность реализации:** S
- 1 сцена + 1 скрипт.
- Пул из 8–10 заказов.
- Простые вероятностные проверки.
- Быстрый одноэкранный flow.

### 4.8. Горящий дедлайн (`deadline`) — Run 4+

**Сатира:** «Три задачи. Два дедлайна. Один ты.»

**Правила:**
- На экране 3–4 задачи с таймерами.
- У каждой задачи:
  - Прогресс-бар (0–100%).
  - Дедлайн (таймер).
  - Важность (1–3 звёзды).
- Игрок распределяет «фокус»:
  - Клик по задаче = работа над ней.
  - Фокус можно переключать.
  - Скорость заполнения прогресс-бара зависит от сложности.

- Если дедлайн наступил, а прогресс < 100% → штраф за провал.
- Если закрыл задачу до дедлайна → бонус за скорость.

**Действия:**
- «Фокус» (клик) — работает над задачей (заполняет прогресс-бар).
- «Делегировать агенту» — агент работает над задачей (медленнее, но параллельно).
- «Отложить» — дедлайн сдвигается, но награда уменьшается.
- «Сдать как есть» — закрыть задачу с текущим прогрессом (меньше денег).

**Сложность реализации:** L
- Много параллельных состояний.
- Таймеры.
- AI-принятие решений.
- Рекомендуется на Run 4+, после того как все остальные мини-игры готовы.

## 8. Психологическая петля

### 1. Первые 30 секунд — Work Hub

Игрок видит список работ. Карточки показывают разную награду и риск. Даже без чтения правил понятно:
- «Сортировка» — скучно, надёжно.
- «Почта» — интригующе.
- «Фикс багов» — страшновато, но награда выше.

Игрок выбирает не по калькуляции, а по любопытству: «А что там в Почте?»

### 2. Выбор — почему не всегда самая прибыльная

- `freelance` даёт до $40, но шанс токсичного клиента может сожрать прибыль и настроение.
- `bugfix` может пойти в каскад ошибок и дать $0.
- `sorting` даёт меньше всего, но надёжно.
- Игрок с 2 энергией не пойдёт в `report` за 3 энергии.

Дилемма: «Быстро и безопасно» vs «Рискованно и много».

### 3. Где появляется риск

- Везде. Но по-разному:
  - В `sorting` — ошибка стоит -$1, нестрашно.
  - В `bugfix` — ошибка плодит новые баги.
  - В `meeting` — можно стать ответственным.
  - В `freelance` — токсичный клиент.
  - В `report` — опасный блок может взорваться.

Риск не бинарный. Это создаёт моменты «блин, я почти закрыл, давай ещё одну попытку».

### 4. Где появляется «ещё одну попытку»

- `freelance`: «В этот раз повезёт с бонусом.»
- `bugfix`: «Сейчас закрою без ошибок.»
- `report`: «Соберу гениальный отчёт.»
- Везде, где есть вариативность результата и частичный контроль игрока.

### 5. Как апгрейды меняют поведение

- До апгрейда: игрок мучается с письмами-ловушками.
- После `spam_filter`: одно письмо подсвечено — «ага, вот это пропустить нельзя».
- До апгрейда: страшно идти в `bugfix` из-за каскада ошибок.
- После `stackoverflow_premium`: один неверный фикс исключён — «теперь можно».

Апгрейды не просто дают +цифру. Они меняют восприятие мини-игры.

### 6. Как ИИ-агенты создают рост

- Агент не просто даёт пассивный доход (это скучно).
- Агент взаимодействует с мини-игрой: решает карточку, сортирует письмо, проверяет результат.
- Агент может ошибиться — и это создаёт **историю**, а не просто раздражение.
- «Почтовый демон перепутал письмо от начальника со спамом» — это смешно.

### 7. Как ошибки агентов делают игру смешнее

- Ошибка агента всегда сопровождается сатирическим комментарием в EventLog.
- Ошибка не убивает прогресс — она стоит денег/энергии, но не ломает день.
- Игрок может перекрыть ошибку агента своими действиями (пойти в Почту сам после провала демона).
- Это создаёт динамику «я лучше чем мой агент» — удовлетворение от собственных навыков.

### 8. Отличие Run 2 от Run 1

| | Run 1 | Run 2 |
|---|---|---|
| Работ | 1 (сортировка) | 3–4 (хаб) |
| Выбор | Нет | Стратегический |
| Апгрейды | 3 базовых | 8 (разные мини-игры) |
| Агенты | 1 (стажёр) | 3+ |
| Длительность дня | 3–5 мин | 5–10 мин |
| Цели | Выжить | Оптимизировать |
| Ощущение | «Учусь» | «Играю» |

### 9. Что оставить на Run 3+

- `deadline` (горящий дедлайн) — сложная механика, не для MVP.
- `report` и `ai_check` — требуют больше контента и тесной связи с агентами.
- `freelance` — простой в реализации, но лучше подать как «продвинутую» работу.
