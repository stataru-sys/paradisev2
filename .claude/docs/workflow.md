# Workflow — keeper и coder в двух чатах

Проект ведётся в **две параллельные роли в разных чатах Claude Code**, чтобы не засорять контекст. Это перенесено из Unity-версии Paradise2033 и хорошо себя показало.

## Keeper-чат (постоянный)

- **Что делает:** дизайн, нарратив, баланс, ревью кода, обновление концепта и docs.
- **Что НЕ делает:** не пишет код. Только текст и решения.
- **На выходе каждого витка:** обновлённый `next.md` для coder'а — что именно делать в следующем слайсе.
- **Живёт долго:** один и тот же чат может работать неделями, потому что не забивается кодом.

## Coder-чат (свежий каждый слайс)

- **Открывается:** командой `claude` в папке `D:\ClaudeProjects\paradise`.
- **Первое действие:** прочитать `.claude/handoff/next.md`.
- **Что делает:** один слайс из плана, коммит, заполняет `.claude/handoff/done.md`.
- **Закрывается:** после коммита и done.md. Не пытаемся «продолжить с того же места» — следующий слайс = новый чат.
- **Почему так:** контекст coder-чата быстро забивается логами компиляции, дампами сцен, скриншотами Godot — на 4-5 слайсе модель начинает «забывать» ранние решения. Свежий чат = чистый контекст + полный набор статики (CLAUDE.md, docs/) загружается заново.

## Передача контекста

| Канал | Кто пишет | Кто читает | Когда |
|---|---|---|---|
| `CLAUDE.md` | keeper (правки) | оба чата | автоматически при старте |
| `.claude/docs/*` | keeper | оба чата | автоматически (через CLAUDE.md или явно) |
| `.claude/plans/*.md` | keeper | оба чата | по ссылке из next.md |
| `.claude/handoff/next.md` | keeper | coder | первое действие coder-чата |
| `.claude/handoff/done.md` | coder | keeper | после коммита слайса |
| Auto-memory `~/.claude/projects/D--ClaudeProjects-paradise/memory/` | оба | оба | автоматически |

## Шаблон next.md

```markdown
# Slice N — <короткое имя>

## Goal
Одно-два предложения: что должно работать к концу слайса.

## Files to touch
- `scripts/...` — что
- `scenes/...` — что
- `resources/...` — что

## Acceptance
- [ ] Конкретный observable критерий 1
- [ ] Конкретный observable критерий 2
- [ ] Тест/play через MCP: <команда или сценарий>

## Out of scope (НЕ делать в этом слайсе)
- ...
```

## Шаблон done.md

```markdown
# Slice N — done

## Что сделано
- ...

## Что НЕ влезло (и почему)
- ...

## Сюрпризы / решения по ходу
- ...

## Что keeper'у проверить
- ...

## Коммит
`<sha>` — "<message>"
```
